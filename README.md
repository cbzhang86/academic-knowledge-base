> 面向公共管理学、社会学领域的学术文献管理 + 系统综述工作流知识库
> 状态：**115** 篇精读报告 · **115** 篇 PDF 原文 · 126 篇跨篇沉淀 · 9 篇方法笔记
> **论文产出**：《技术在场、服务缺席》（13,135 字）— 已通过 ARS 全流程管线（审稿→修订→定稿）

## 概述

本知识库的目标是从**论文检索→精读分析→跨篇沉淀→报告输出**全流程支撑学术研究。
我们在 5 个方向（公共管理学、社会学、老龄化研究、青少年研究、交叉研究）持续追踪中文 CSSCI 论文，
将零散的 PDF 转化为结构化知识资产，支持选题、开题、系统综述和论文写作。

### ⭐ 架构特色：知识注入管线

本知识库最独特的设计是**将跨篇沉淀直接注入 AI 论文写作管线**：

```
知识库精读沉淀（理论框架·关键论点·研究空白）
     │
     ▼
 ars_kb_bridge.py（按ARS阶段切分知识包）
     │
     ├──→ 引文对照包 → 完整性审核（Stage 2.5）
     ├──→ 审稿背景包 → 同行评审（Stage 3）
     ├──→ 理论推荐包 → 修订（Stage 4）
     └──→ 引文审计包 → 定稿（Stage 5）
```

**核心思想**：知识不只服务于人读，也为AI写论文时可用。每个阶段启动前生成知识包，让审稿/修订/定稿 Agent 在上下文中拥有你积累的全部精读知识，实现"不会因写作而脱离已有积累"。

这不是 API 集成，而是 **Prompt 上下文注入模式（Knowledge Injection Pattern）**——将结构化知识翻译成 LLM 在各学术写作阶段最需要的输入格式。

### 核心流水线

```
01_论文原文/  (PDF 原材料)
     ↓ 逐篇精读分析
02_精读报告/  (v10 格式，7 维评分 + 4 项思考)
     ↓ 脚本自动提取
03_学术写作素材库/  (摘要·理论框架·关键论点·研究空白)
     ↓ 日报/周报/月报
05_报告/  (定期产出，全局进展)
```

## 知识加工流

### 第 1 步：采集
通过多源通道（ncpssd 中文 CSSCI、arXiv、OpenAlex），按关键词池搜索并下载论文 PDF。
- 自动去重（本地 SQLite `config/dedup.db`，MD5 指纹）
- PDF 按方向分类存入 `01_论文原文/`
- 可选的 Zotero 归档后端

**对话示例：**
```
你：
老龄化方向很久没更新了，帮我搜索一下新论文。

我：
从关键词池中加载老龄化的关键词，
启动 ncpssd 搜索"养老服务 数智化"，获取结果列表，
去重（排除已有 PDF 和精读），筛选出相关论文，
下载 PDF 并存入 01_论文原文/老龄化/，
更新检索日志，更新进度看板。

你（查看后）：
有篇蒲新微2026的论文看起来不错，下载了。帮我继续搜"代际反哺"。

我（继续搜索）：
启动 OpenAlex 搜索"代际反哺 数字鸿沟"，
去重检查，找到1篇相关论文，
下载 PDF 并标准化文件名，记录到检索日志。
```

### 第 2 步：精读（⭐核心）
每篇论文按 `analysis_template.md`（v10 格式）逐篇分析，175-205 行，包含：
1. **YAML frontmatter** — 标题/主题/方法/变量/发现/关联性
2. **论文信息表** — 作者/期刊/DOI/CSSCI 等完整元数据
3. **研究概述** — 一句话定位 + 理论框架 + 研究方法
4. **7 维质量评分** — 每维必须有评价性理由（非单纯数字）
5. **内容拆解** — 核心发现 + 引文逐字匹配 + 概念定义
6. **我的思考** — 最有启发的点 / 可借鉴之处 / 待验证 / 与我研究关联
7. **标签 + 元信息**

**对话示例：**
```
你：
新下载了3篇老龄化论文，今天来精读。

我：
依次读取 01_论文原文/老龄化/ 中未精读的 PDF，
按 v10 模板逐篇撰写精读分析（175-205行），
每篇写完提交审核，你确认后继续下一篇。

你（审核第1篇后）：
第1篇没问题，继续写第2篇。

我（继续）：
撰写第2篇精读报告，
提交审核，等待确认。
```

> ⚠️ 铁律：不准简写 / 不准跳过审核 / 引文必须逐字匹配原文
> ⚠️ 铁律2：入库路径必须按 `| **入库路径** | \`01_论文原文/方向/文件名.pdf\` |` 完整格式
> ⚠️ 铁律3：报告存放目录必须与PDF目录一致
> ⚠️ 铁律4：文件名必须含年份_作者前缀（如 `2026_张三_标题_精读报告.md`）

### 第 3 步：跨篇沉淀
`regenerate_all.py` 自动扫描全部精读报告，提取：
- **摘要** — 各方向核心发现段聚合（126 篇素材）
- **理论框架** — 524 个理论概念 + 交叉引用索引
- **关键论点** — 跨篇引用汇编（86 篇，带来源标注）
- **研究空白** — 各方向不足与展望段聚合（选题原料）

### 第 4 步：论文写作（ARS 管线 + 知识注入）

本知识库集成了 **ARS（academic-research-skills）论文写作管线**。与普通 ARS 使用不同，你的每一步都植入了知识库积累：提交初稿 → 我分析方向 → 逐阶段生成知识包 → 注入上下文 → 输出来自你的真实文献积累。

**对话示例（推荐——这是最自然的交互方式）：**

```
你：
请帮我审核这篇论文初稿。
路径：10_研究输出/定稿/技术在场服务缺席.md

我（第1步：分析→生成审稿背景包→输出审核）：
先读取论文，判断方向为"养老服务 数字排斥"，
然后运行 ars_kb_bridge.py stage-3 --topic "养老服务 数字排斥"，
将知识库中相关理论框架、研究空白、关键论点打包，
以审稿人身份结合知识库背景进行全面评审，
输出5份审稿报告（理论框架/方法/论证/创新/写作）。

你（阅读报告后）：
审稿意见收到了。请做引文审计。

我（第2步：提取引文→生成引文审计包→输出审计）：
从论文参考文献段提取引文列表，
运行 ars_kb_bridge.py stage-5 --refs "蒲新微,刘志军,..."，
对照知识库115篇精读，检查引文覆盖情况，
输出已引用 vs 漏引的高评分文献对照表。

你（确认后）：
请根据审稿意见和引文审计，给出修订方案。

我（第3步：综合审稿+引文审计→修订建议）：
综合前面两步的发现，
输出按重要性排序的修订方案（每项: 原文位置+修改方案+预期效果）。
```

**知识注入原理**：每一步启动前运行 `ars_kb_bridge.py` 生成知识包 → 知识包和上一步输出一起传给下一步 → AI 在每一步的上下文中拥有你积累的真实文献知识。

**中文学术格式输出**：
```bash
python "14_工具脚本/report/create_formatted_docx.py" convert \
  -i 终稿.md -o 终稿_学报版.docx --style 学报
```

> ⚠️ 如果走完整 ARS 全流程：Claude Code 需锁定 v2.1.165
> 各阶段脚本命令详见 `CLAUDE.md § 阶段7`

### 第 5 步：报告产出
- **日报** — 今日新报告 + 进度 + 明日计划
- **周报** — 本周全局 + 缺口分析 + 方法建议
- **月报** — 资产盘点 + 健康评估 + 研究空白趋势

## 分支说明

此仓库包含两个分支：

| 分支 | 内容 | 适用场景 |
|------|------|---------|
| **master** | 完整知识库（115 篇精读 + 115 PDF + 全套工具） | 你的日常研究 |
| **framework** | 纯技能框架（skills/ + config/ + docs/，不含知识内容） | 其他人 fork 后换领域使用 |

切换到 framework 分支即可获得可复用的管线框架，只需替换 `config/areas.yaml` 中的关键词即可用于任何学科。
